# Phase 6 종료 보고서 (phase-06-report.md)

* **단계명**: Phase 6 - RAG QA 파이프라인 구현
* **작업 기간**: 2026-06-10
* **작성자**: Antigravity (AI Coding Assistant)

---

## 1. 구현 개요
* **목표**: 유사도 검색(Retrieval) 결과를 기반으로 OpenAI `ChatModel`을 연동해 사용자 질문에 근거한 정확하고 정직한 RAG 답변을 생성하고, 질문과 답변을 함께 반환하는 REST API 종단 파이프라인을 구축했습니다.
* **MVP 및 YAGNI 원칙 적용 사항**:
  * **프레임워크 오버헤드 배제**: Spring AI의 고수준 Advisor 프레임워크(`RetrievalAugmentationAdvisor` 등)를 도입하는 대신, 주입받은 `ChatModel`을 명시적이고 단순하게 직접 기동하도록 코드를 작성해 디버깅 오버헤드를 억제했습니다.
  * **프롬프트 관리 단순화**: 프롬프트 외부 로더 컴포넌트 생성을 배제하고, `RagService` 내부에 `private static final String` 상수로 선언하여 코드 가독성과 단순성을 극대화했습니다.
  * **API DTO 슬림화**: 불필요한 출처 반환(Citation UI 대상 제외)을 제거하고, 호출 이력 추적 및 질의 검증에 필수적인 질문(`question`)과 답변(`answer`)만 반환하는 `RagResponse` DTO를 정의했습니다.

---

## 2. 검색 결과 부재 시 조기 우회 반환 로직

* **배경**: 지식 검색 결과가 빈 리스트(`chunks.isEmpty()`)인 경우, LLM API를 호출하여 요금을 소모하고 불필요한 네트워크 대기시간을 유발할 이유가 없습니다.
* **설계**: `RagService`에서 유사 청크가 없을 때, OpenAI API call 호출 없이 즉시 응답을 반환하도록 구현하여 불필요한 LLM 호출을 방지했습니다.

## 3. RAG 프롬프트 지침 (환각 감소 설계)

LLM의 매개변수 지식으로 거짓 답변을 지어내거나 유추하는 행위를 완전히 차단하기는 불가능하므로, 환각 가능성을 낮추기 위해 문맥 기반 답변 지침을 프롬프트에 포함하여 설계했습니다.

```java
private static final String SYSTEM_INSTRUCTION = 
    "당신은 문서 지능 플랫폼의 RAG QA 전문 어시스턴트입니다.\n" +
    "아래 제공되는 'Context' 안의 정보만을 사용하여 사용자의 'Question'에 친절하고 정확하게 답변하십시오.\n" +
    "만약 제공된 Context 내용과 사용자의 질문 내용 사이에 아무런 연관성이 없거나 답변할 근거가 없다면, " +
    "절대 답변을 지어내지 말고 반드시 다음과 같이 답변하십시오:\n" +
    "'제공된 문서 내에 해당 질문에 답변할 수 있는 관련 정보가 존재하지 않습니다.'";
```

## 4. 검증 및 테스트 결과

### **4.1 RAG 동작 검증 시나리오 (실 작동 검증)**

실제 RAG 파이프라인의 검색 정합성과 자연어 답변 매칭을 검증하기 위해 수행된 대표적인 실동작 테스트 시나리오는 다음과 같습니다.

* **시나리오 A: 문서 정보 기반 질의 (성공 케이스)**
  * **질문**: "환불 정책은 어떻게 되나요?"
  * **Retrieval 결과 (Top-K Chunks)**:
    * `[Chunk 1] 환불은 구매 후 7일 이내에만 가능합니다. 단, 개봉 후 파손된 상품은 환불이 불가합니다.`
  * **LLM 최종 답변**:
    * `"환불은 구매 후 7일 이내에만 가능합니다. 다만, 개봉 후 파손된 상품은 환불이 불가능하니 유의하시기 바랍니다."` (컨텍스트 기반으로 자연스러운 대답 생성 성공)

* **시나리오 B: 문서 외 범주의 질의 (우회 케이스)**
  * **질문**: "연차 정책과 휴가 일수는 어떻게 되나요?"
  * **Retrieval 결과 (Top-K Chunks)**:
    * `[검색 결과 없음]` (유사도 기준을 만족하거나 문서에 관련 청크가 존재하지 않음)
  * **LLM 최종 답변**:
    * `"제공된 문서 내에 해당 질문에 답변할 수 있는 관련 정보가 존재하지 않습니다."` (외부 지식에 의존한 거짓 정보 주입 차단 및 조기 우회 정상 동작)

### **4.2 단위 테스트 및 빌드 결과**
* **`RagServiceTest.java`**: 
  * `askQuestion_Success`: 관련 청크 문맥이 획득되었을 때 프롬프트를 구성해 `ChatModel.call(prompt)`를 호출하고 LLM 생성 답변이 성공적으로 DTO에 바인딩되는지 검증 완료.
  * `askQuestion_BypassWhenNoChunks`: 검색 결과가 부재할 때 `ChatModel`을 기동하지 않고 조기 우회 반환 문장이 즉각 리턴되며, `verifyNoInteractions(chatModel)`를 통해 외부 API 결합이 성공적으로 차단됨을 입증 완료.
* **빌드 안정성 검증**:
  * Spring AI 1.0.0-M6 규격상 `AssistantMessage`에서 본문 문자열을 가져올 때 `.getContent()`가 아닌 `.getText()`를 호출해야 하는 타입 호환성 이슈를 완벽히 해결하고 `./gradlew build` 컴파일 및 테스트 **성공(BUILD SUCCESSFUL)**을 완료했습니다.
