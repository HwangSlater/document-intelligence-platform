# 프로젝트 스토리

## 프로젝트 시작 배경

실무 환경에서는 수많은 문서가 생성된다.

예시

* 규정집
* 기술문서
* API 문서
* 회의록
* 운영가이드

하지만 사용자는 필요한 정보를 찾기 위해 직접 문서를 탐색해야 한다.

문서가 증가할수록 정보 탐색 비용 역시 증가한다.

---

# 문제 정의

기존 문서 검색은 다음과 같은 문제가 존재한다.

## 키워드 검색

* 정확한 단어를 알아야 함
* 의미 기반 검색 불가능

---

## 문서 직접 탐색

* 시간 소모
* 생산성 저하

---

## 일반 LLM

* 내부 문서 접근 불가
* 환각 발생 가능

---

# 해결 전략

문서를 벡터화하여 저장하고

사용자 질문 시

관련 문서를 검색한 후

LLM이 답변하도록 설계하였다.
