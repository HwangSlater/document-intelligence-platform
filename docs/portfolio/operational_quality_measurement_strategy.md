# 📊 RAG QA & Retrieval 로컬 품질 측정 전략서
(Local Quality & Retrieval Performance Measurement Strategy)

본 문서는 사용자가 수집한 **9개의 실제 PDF 전문 문서**를 로컬 환경(Docker Compose PostgreSQL + 로컬 Spring Boot 실행)에 적재하고, RAG 및 정보 검색(Information Retrieval) 품질을 정밀 계측하여 포트폴리오와 면접의 실증력을 극대화하기 위한 **로컬 품질 측정 및 관측성(Observability) 전략서**입니다.

---

## 1. 실데이터 기반 데이터셋 & 품질 평가지표 구성

### 1.1. 로컬 실데이터 수집 범위
* **대상 문서**: 경남대 대학원 학칙, 가명정보 가이드라인, SW 공급망 보안 가이드라인 등 총 9개 실제 문서.
* **청크 구축 규모**: **150 ~ 300개** (500 tokens 크기, 100 글자 중첩 기준 분할).
* **평가 셋 규모**: **Gold Standard Test Set 총 20쌍** 구축 (사용자 질문 - 기대 청크 ID - 기대 답변 구조).

### 1.2. 정보 검색(Retrieval) 평가 지표 (Top-K = 5 기준)
1. **MRR (Mean Reciprocal Rank - 평균 역수 순위)**
   * 유사도 검색 시 사용자가 기대한 정답 문서 청크가 몇 번째에 정렬되어 노출되는지를 평균 역수 비율로 계산.
2. **Recall@5 (재현율)**
   * 20개 질문별 필수 정답 청크들이 Top-5 결과 묶음 안에 최종적으로 누락 없이 포함된 비율.
3. **Hit Rate (적중률)**
   * 검색된 Top-5 중 정답 청크가 적어도 1개 이상 존재하여 LLM에게 성공적으로 힌트를 제공한 비율.

## 4. 로컬 모니터링 및 계측 연동 (Observability)

로컬에서 RAG 시스템을 구동할 때도 Micrometer와 Actuator Prometheus 엔드포인트를 열어 JVM 힙 메모리 및 쿼리 실행 속도를 감시합니다.

### 4.1. Micrometer 커스텀 메트릭 트래킹 연동
로컬 스프링 부트 애플리케이션의 `/actuator/prometheus` 엔드포인트에 계측용 데이터가 정상 빌드되도록 아래의 `RagMetricsTracker` 컴포넌트를 연동해 둡니다.

```java
// Spring AI / RAG 지연 시간 및 카운터 계측용 Registry 바인딩
Timer.builder("rag.retrieval.latency").register(meterRegistry);
Timer.builder("rag.generation.latency").register(meterRegistry);
Counter.builder("rag.requests.bypass").register(meterRegistry);
```

### 4.2. 로컬 그라파나 대시보드 시각화 (PromQL)
로컬에 Prometheus 및 Grafana 컨테이너를 함께 띄울 경우 가시화할 패널 구성입니다.
* **Retrieval p95 Latency**: `histogram_quantile(0.95, sum(rate(rag_retrieval_latency_seconds_bucket[1m])) by (le))`
* **LLM Chat Latency**: `histogram_quantile(0.95, sum(rate(rag_generation_latency_seconds_bucket[1m])) by (le))`
* **OpenAI API 누적 비용 산출**: `sum(rag_openai_cost_dollars_total)`
